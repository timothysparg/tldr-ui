= TLDR UI Extensions
:toc:
:toclevels: 2

This UI ships two separate extension entrypoints:

* An Antora extension that collects article metadata into `site.posts`
* An AsciiDoc extension that customizes code block rendering, injects static icons, and enables the native TLDR admonition at build time

The Antora extension automatically registers and configures the AsciiDoc extension, simplifying your playbook setup.

== Extension Overview

[cols="1,1,3"]
|===
|Type |Source entrypoint |Purpose

|Antora
|`extensions/antora-tldr-ui.js`
|Collect articles into `site.posts` and auto-register the AsciiDoc extension

|AsciiDoc
|`extensions/asciidoc-tldr-ui.js`
|Apply Shiki highlighting, inject code headers, download icons, and enable the `[TLDR]` admonition
|===

== Bundled Artifacts

`gulp build` and `gulp bundle` produce bundled CommonJS extension files in `build/extensions/`:

* `build/extensions/antora-tldr-ui.js`
* `build/extensions/asciidoc-tldr-ui.js`

Place *both* files in your project's `extensions/` directory to use them.

== Registering The Extensions

Register only the main Antora extension; it handles the AsciiDoc setup internally:

[source,yaml]
----
antora:
  extensions:
    - require: ./extensions/antora-tldr-ui.js
      shiki:
        lightTheme: github-light
        darkTheme: github-dark
      asciidoc:
        direct_icon_urls:
          pkl: https://raw.githubusercontent.com/apple/pkl-lang.org/refs/heads/main/src/supplemental-ui/img/favicon.svg
          toml: https://raw.githubusercontent.com/toml-lang/toml/refs/heads/main/logos/toml.svg
----


== Antora Extension

The Antora extension:

* scans pages for `:page-role: article`
* collects article metadata
* sorts articles by date, newest first
* injects the result into `site.posts`

=== Required Article Attributes

[cols="1,3"]
|===
|Attribute |Description

|`:page-role:`
|Must be set to `article` for the page to be collected

|`:page-date:` (or `:date:` or `:revdate:`)
|Publication date used for sorting. Format: `YYYY-MM-DD`

|`:description:` (or `:page-description:`)
|Summary text shown in article listings
|===

=== Optional Article Attributes

[cols="1,3"]
|===
|Attribute |Description

|`:featured:`
|Set to `true` to mark the article as featured

|`:image:` (or `:page-image:`)
|Thumbnail image path

|`:navtitle:`
|Override the page title in listings
|===

=== Example Article

[source,asciidoc]
----
= Getting Started with Antora
Author Name
:page-role: article
:page-date: 2026-01-07
:description: Learn how to set up your first Antora documentation site
:featured: true
:image: _images/antora-logo.svg

This is your article content...
----

== TLDR Admonition

The suite includes a native `[TLDR]` block processor that renders a modern, blue-themed summary card.

=== Usage

[source,asciidoc]
----
[TLDR]
====
A brief, high-impact summary of the content below.
====
----

The block is rendered as an `admonition` with the name `tldr`.
The UI bundle includes specific styling and a custom avatar for this block.

== AsciiDoc Extension

The AsciiDoc extension registers a custom HTML5 converter that:

* customizes `listing` blocks with titles or languages
* customizes console-style `literal` blocks
* applies Shiki syntax highlighting at build time
* enables Shiki transformer annotations for richer code examples
* injects `code-header` markup during conversion
* registers the `TLDR` block processor
* downloads missing SVG icons into `build/devicons/`
* fails the build if required icons cannot be downloaded


=== Shiki Highlighting

The converter uses Shiki to highlight source blocks at build time.
That means the generated HTML already contains the highlighted markup and no client-side syntax highlighter is needed.

The Antora extension initializes the shared Shiki highlighter during site generation.
The preview pipeline does the same before converting the preview pages, so preview and Antora builds use the same highlighting path.

The converter enables these Shiki transformer features:

* notation diff
* line highlight
* word highlight
* focus
* error / warning / info highlighting
* optional whitespace rendering
* optional indent guides

You can override the Shiki themes through the Antora extension config:

[source,yaml]
----
antora:
  extensions:
    - require: ../build/extensions/antora-tldr-ui.js
      shiki:
        lightTheme: github-light
        darkTheme: github-dark
----

=== Icon Configuration

The converter uses Devicon by default. For languages not available in Devicon, provide direct download URLs in the playbook under `direct_icon_urls`.

[source,yaml]
----
asciidoc:
  extensions:
    - require: ../build/extensions/asciidoc-tldr-ui.js
      direct_icon_urls:
        pkl: https://raw.githubusercontent.com/apple/pkl-lang.org/refs/heads/main/src/supplemental-ui/img/favicon.svg
        toml: https://raw.githubusercontent.com/toml-lang/toml/refs/heads/main/logos/toml.svg
----

The downloaded icon cache is stored in `build/devicons/`.

=== Supported Block Patterns

The converter adds headers for:

* source/listing blocks with a language
* titled listing blocks
* console-style literal blocks

Example source block:

[source,asciidoc]
....
[source,bash]
----
echo "hello"
----
....

Example titled listing block:

[source,asciidoc]
....
.Dockerfile
[source,docker]
----
FROM node:22
----
....

Example annotated block:

[source,asciidoc]
....
[source,ts]
[code.add=2,code.remove=1]
----
console.log('old')
console.log('new')
console.log('same')
----
....

Example block attributes:

[source,asciidoc]
....
[source,json]
[code.highlight="3",code.word="hello"]
----
{
  "status": "ok",
  "message": "hello world",
  "count": 3
}
----
....

Example block with callouts:

[source,asciidoc]
....
[source,clojure]
[code.callout=4]
----
(defn add [a b]
  (+ a b))

(println (add 2 3))
----
<1> Calls the `add` function and prints the result.
....

Optional block attributes:

* `code.add="2,4"` / `code.remove="3"`
* `code.highlight="1,3-4"`
* `code.focus="2"`
* `code.warning="2"` / `code.error="3"` / `code.info="4"`
* `code.word="term,other"`
* `code.whitespace=true` or `code.whitespace=boundary`
* `code.indent-guides=true`
* `code.indent="4"`
* `code.callout="4"` or `code.callout="5,7,16"` — inject native callout markers before parse (quote the value when specifying multiple lines)

== Build Integration

`gulp build` and `gulp bundle` both bundle the extension entrypoints into `build/extensions/`.

`gulp preview` and `gulp preview:build` register the AsciiDoc extension from the source tree for local preview rendering.
