=== Shiki Transformer Examples

These examples demonstrate the build-time Shiki transformer support used by this UI.

==== Diff Notation

Use `code.add` and `code.remove` block attributes to mark added and removed lines without adding inline comment markers to the snippet.

[source,ts]
[code.add="2,4",code.remove=1]
----
console.log('old')
console.log('same')
console.log('same')
console.log('new')
----

Line 1 is styled as removed, and lines 2 and 4 are styled as added.

==== Line Highlighting

Use `code.highlight` to emphasize specific lines without changing the visible code.

[source,js]
[code.highlight=2]
----
console.log('normal')
console.log('highlighted')
console.log('normal')
----

Only the middle line is highlighted.

==== Focused Lines

Use `code.focus` to dim surrounding lines and draw attention to the important one.

[source,js]
[code.focus=2]
----
console.log('background')
console.log('focus me')
console.log('background')
----

The focused line stays fully emphasized while the surrounding lines are dimmed.

==== Severity Levels

Use `code.warning`, `code.error`, and `code.info` to communicate the intent of specific lines.

[source,js]
[code.warning=2,code.error=3,code.info=4]
----
const ok = true
const needsAttention = false
throw new Error('boom')
console.info('heads up')
----

The second, third, and fourth lines receive warning, error, and informational styling respectively.

==== Word Highlighting

Use `code.word` to highlight a specific word or token inside an otherwise normal line.

[source,js]
[code.word=Hello]
----
console.log('Hello World')
----

Only the `Hello` token is highlighted.

==== Meta-Driven Highlighting

Formats that do not support inline comments can use the same `code.*` block attributes.

[source,json]
[code.highlight=3,code.word=hello]
----
{
  "status": "ok",
  "message": "hello world",
  "count": 3
}
----

The `hello` word is highlighted through `code.word`, and line 3 is highlighted through `code.highlight`.

==== Whitespace Rendering

Use `code.whitespace` to render visible whitespace boundaries.

[source,python]
[code.whitespace=boundary]
----
def greet(name):
    if name:
        print("hi", name)
----

Boundary whitespace markers make indentation changes easier to spot.

==== Indent Guides

Use `code.indent-guides` to render indentation guides for nested blocks.

[source,python]
[code.indent-guides=true]
----
def greet(name):
    if name:
        print("hi", name)
----

Indent guides visually connect nested levels without adding whitespace markers.

==== Console Copy Extraction

Literal console blocks still render with code headers, and copy extracts the shell commands rather than the prompt characters.

....
$ npm ci
$ npm run build \
  -- --watch
....

The visible `$ ` prompt is not included in the copied value, and the continued command is flattened for copying.
